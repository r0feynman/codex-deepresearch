일일 개발 루틴

  1. Ready에서 하나 선택. 가장 중요한 작업만 고릅니다.
  2. In Progress로 이동. 착수한 이슈만 작업 중으로 표시합니다.
  3. Codex에게 구현 요청. Issue scope와 Acceptance Criteria를 기준으로 맡깁니다.
  4. 검증. 테스트, 산출물, 체크리스트를 확인합니다.
  5. PR 또는 커밋. 변경 내용을 리뷰 가능한 단위로 묶습니다.
  6. Done으로 이동. 머지 또는 이슈 종료 후 완료로 표시합니다.
